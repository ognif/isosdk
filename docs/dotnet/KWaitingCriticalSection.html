Remarks

Logics of this class is like the same of using the simple KCriticalSection. Main thread should create an instance of this class. And, for example, two (concurenting) threads: Thread A and Thread B. Let's imagine that during it's work Thread A should check for some condition (value of a global variable) and only if the condition is met it can continue execution and Thread A should wait for Thread B , which can influe on the condition, otherwise. In this situation Thread A should 1) Lock(), 2) Check the condition, 3) Wait() (if needed). Wait() Unlock()s on it's start. This allows other threads to enter the same critical section. Wait() Lock()s on it's finish. 4) Unlock(). 

Thread B should 1) Lock(), 2) Influe on the condition, 3) If the condition was met it should Signal() to Wait()ing Thread A or BroadcastSignal() if many threads are in waiting state, 4)Unlock. Note that it's your responsibility to: 1) Lock() [Unlock()] before [after] calling Wait() or Signal(). 2) Check if Lock() and Unlock() returned successive result. 3) Call [Broadcast]Signal() only if some process is Wait()ing.
