Members
Members 
Description 
TCHAR chVolumeLabel[128]; 
This argument sets the volume name of the medium (null terminated). This name will be displayed e.g. in the Windows Explorer. This string may neither contain high characters nor spaces. 
int32 nWriteMethod; 
This parameter sets the burning method. This option is only allowed for CD-R/CD-RW media. For DVD media it is ignored. Values are:
BS_WM_TAO,
BS_WM_DAO,
BS_WM_DAO96 
BS_BOOL bJolietFileSystem; 
If you want to use the Joliet file system, pass BS_TRUE to activate it and pass BS_FALSE to deactivate it. This option is only available for data projects. 
BS_BOOL bBootable; 
If you want to make a bootable device, pass BS_TRUE to activate it and pass BS_FALSE to deactivate it. This option is only available for data projects. 
TCHAR chBootImage[_MAX_PATH]; 
File name of the image (with path information) 
BS_BOOL bFinalize; 
This property states whether the medium should be finalized. BS_TRUE finalizes the medium, whereas BS_FALSE burns a multi-session medium. 
BS_BOOL bTestBurn; 
Get/Set the simulation option. The value "BS_TRUE" activates the burning simulation and false skips the simulation.
The IsoSDK will make a dry run with this option. To use the final write to the disc you have to set this value to false. 
BS_BOOL bPerformOPC; 
DVD writing devices perform a test write and read in an area inside of the lead-in, in order to determine the best laser power for recording. This allows the device to adjust to each disc, which may vary slightly from different manufacturers, or for other reasons.

BS_TRUE/BS_FALSE whether you want to perform OPC or not. Pass true to activate it and pass false to de-activate it. 
BS_BOOL bVerifyAfterBurn; 
This argument enables or disables the verify after burning. This option is only available for Data CD/DVD/BD ISO and UDF projects. 
int32 nCacheSize; 
Sets the size of the burning buffer to be used (in bytes; default: 4 * 1024 * 1024 [4 MB])
We recommend to use a range between 4 MB and 32 MB.
4 MB = CD
16 MB = DVD
32 MB = BD 
BS_BOOL bUnderrunProtection; 
You can activate the buffer-underrun-protection by setting the value to BS_TRUE. BS_FALSE de-activates the buffer protection. 
BS_BOOL bEjectAfterBurn; 
This argument determines whether the medium will be ejected after the burning process. BS_TRUE ejects the medium, and BS_FALSE prevents this. 
BS_BOOL bAutoErase; 
This argument sets the autoerase function for CD/DVD/BluRay ReWriteable. If this option is set to BS_TRUE the FoxBurner SDK will erase a RW before the new burnjob started. Please note that this option use the "Quick erase" function. 
BS_BOOL bRockRidgeFileSystem; 
This argument disable or enable the RockRidge support. Default value is BS_FALSE. This option is supported in following projects: Data, ISO/UDF, Mixed Mode. 
int32 nCopies; 
This argument sets the number of copies to burn. Only available in GUI mode. 
BS_BOOL bPadDataTracks; 
This option will pad the data track of a disc to the min. size of 4 seconds. This is important for Data, Multisession and Mixed Mode disc if your work with very small sizes. 
Description

IsoSDK sub class that is used to get and set the parameters for the current project.
